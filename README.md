## What is GreenSense?

GreenSense takes 5 inputs from the user — room type, light level, climate, maintenance preference, and whether they have pets — and runs them through a 4-stage AI decision pipeline to return a ranked list of compatible plants. Each result includes a composite score, natural language explanation, care guide, survival probability, and nearby nursery locations with pricing.

---

## Features

- **Smart Filtering** — hard-eliminates incompatible plants based on light, climate, and pet safety before scoring
- **ML-Tuned Scoring** — 7-criteria scoring engine driven by weights learned from a Decision Tree trained on 3,000 samples (R² = 0.947)
- **Natural Language Explanations** — per-plant reasoning generated by the ExplanationEngine
- **Survival Probability** — percentage score shown as an animated ring on each plant card
- **Score Breakdown** — horizontal bar chart for all 7 scoring criteria per plant
- **Find Nearby Nurseries** — dedicated page showing 6 Pune nurseries with average price per plant, address, contact, hours, and directions
- **Live Filter Counter** — form shows compatible plant count in real time as preferences change
- **Light Theme UI** — fully light-themed React frontend across all 5 pages

## OOP Concepts Demonstrated

| Concept | Where | How |
|---|---|---|
| **Encapsulation** | `Plant.java`, `UserInput.java`, all models | All fields are `private`. External code accesses data only through getters. |
| **Abstraction** | `EngineStage<I,O>` interface | Single `process()` method hides all engine implementation details. |
| **Polymorphism** | `FilterEngine`, `ScoringEngine`, `ExplanationEngine` | All three implement `EngineStage`. Treated uniformly by `DecisionEngine`. |
| **Composition** | `DecisionEngine.java` | Owns instances of all 3 engines. Orchestrates without inheriting from any. |
| **Comparable** | `ScoredPlant.java` | Implements `Comparable<ScoredPlant>` for natural descending sort via `Collections.sort()`. |
| **Single Responsibility** | All engine classes | Every class has one job — FilterEngine only filters, ScoringEngine only scores. |
| **Dependency Injection** | `ScoringEngine` ← `ScoringWeights` | Weights are constructed externally and injected via the constructor. |
| **Open/Closed Principle** | `ScoringEngine` + `MLWeightsLoader` | `ScoringEngine` never changed when ML was introduced — only the injected weights changed. |

## ML Integration

GreenSense uses a **scikit-learn Decision Tree Regressor** to learn optimal scoring weights from data rather than relying on hardcoded values.

**How it works:**

```
train_weights.py
    ↓ generates 3,000 synthetic user-plant interaction samples
    ↓ trains DecisionTreeRegressor (max_depth=6, R²=0.947)
    ↓ extracts feature importances
    ↓ normalizes to 100-point scoring budget
    ↓ writes ml_weights.json

Java startup
    ↓ MLWeightsLoader.load() reads ml_weights.json from classpath
    ↓ builds ScoringWeights object with ML-learned numbers
    ↓ injects into ScoringEngine via constructor
    ↓ every recommendation now uses ML-tuned weights
```

**The model runs once (training). At runtime, Java just reads the weights from JSON — no Python required to use the app.**

If `ml_weights.json` is missing, `MLWeightsLoader` silently falls back to `ScoringWeights.defaults()`.

## How to Run

### Step 1 — Train the ML model (first time only)

```bash
cd Greensense/greensense_java
pip install scikit-learn numpy
python train_weights.py
```

This generates `src/main/resources/ml_weights.json` automatically.

> Skip this step if `ml_weights.json` already exists in `src/main/resources/`.

---

### Step 2 — Start the Java backend

```bash
cd Greensense/greensense_java
mvn spring-boot:run
```

Wait for:
```
Started GreenSenseApplication in X.X seconds
```

Backend runs at **http://localhost:8080**

---

### Step 3 — Start the React frontend

Open a **second terminal**:

```bash
cd Greensense/greensense_clean
npm install
npm run dev
```

Open **http://localhost:5173** in your browser.

---

### Both terminals must stay open while using the app.

> If the Java backend is not running, the frontend automatically falls back to a client-side scoring simulation built into `decisionEngine.js`.

---

### Port conflict?

If port 8080 is already in use, add this to `greensense_java/src/main/resources/application.properties`:

```properties
server.port=8081
```

Then update the API base URL in `greensense_clean/src/engine/decisionEngine.js` to match.

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/api/recommend` | Main endpoint. Takes `UserInput` JSON, returns ranked `RecommendationResult` list |
| `POST` | `/api/compatible-count` | Returns count of plants that pass the filter for current preferences |
| `GET` | `/api/total-plants` | Returns total number of plants in `PlantKnowledgeBase` |

**Example request body for `/api/recommend`:**
```json
{
  "roomType": "living_room",
  "light": "medium",
  "climate": "tropical",
  "maintenance": "low",
  "hasPets": true
}
```
